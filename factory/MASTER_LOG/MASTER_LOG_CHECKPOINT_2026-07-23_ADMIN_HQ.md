### 자동 진입 대상

신뢰된 기기로 등록된 아래 두 기기만 로그인 없이 `/admin`에 바로 진입한다.

2. 내 휴대폰

### 로그인 필요 대상

- 회사 PC
- PC방

- 등록되지 않은 모든 기기

## 3. 신뢰된 기기 방식 확정

IP 고정 방식이 아니라 **Trusted Device(신뢰된 기기)** 방식으로 구현한다.

이유:

- 가정용 인터넷 IP가 바뀌어도 자동 로그인이 유지된다.
- 휴대폰과 컴퓨터를 각각 안전하게 등록할 수 있다.
- 다른 기기는 자동으로 차단할 수 있다.
- 기기 교체 시 기존 기기를 삭제하고 새 기기를 등록할 수 있다.

### 보안센터에서 제공할 기능

- 신뢰된 기기 목록
- 기기 이름
- 등록일
- 최근 접속일
- 현재 기기 등록
- 기기 삭제
- 모든 기기 로그아웃

초기 표시 이름:

- `내 컴퓨터`
- `내 휴대폰`

## 4. 휴대폰 QR 등록 방식 확정

휴대폰은 `/admin` 보안센터에서 만든 QR코드를 촬영해 신뢰된 기기로 등록한다.

### 등록 흐름

1. 신뢰된 PC에서 `/admin` 접속
2. 보안센터 진입
3. `휴대폰 등록(QR)` 선택
4. QR코드 생성
5. 휴대폰 카메라로 QR 촬영
6. 휴대폰에서 등록 확인
7. 서버가 일회용 토큰 검증
8. 해당 휴대폰을 신뢰된 기기로 저장
9. 이후 휴대폰에서 `/admin` 자동 진입

### QR 보안 규칙

- 1회만 사용 가능
- 5분 후 자동 만료
- 새 QR 생성 시 이전 QR 즉시 폐기
- QR에는 관리자 비밀번호를 넣지 않는다.
- 서버가 서명한 일회용 등록 토큰만 넣는다.
- 등록 완료 후 토큰 즉시 폐기
- 이미 사용된 QR 재사용 금지
- 만료되거나 위조된 토큰은 등록 거부

## 5. Lina AI 보안 역할

향후 Lina AI가 보안센터의 사용자 인터페이스 역할도 수행하도록 설계한다.

예시 명령:

- `리나야, QR 만들어줘.`
- `리나야, 이 기기 삭제해.`

단, Lina의 대화 명령만으로 보안 검증을 우회하지 않는다. 실제 등록·삭제는 서버 보안 절차를 거친다.
