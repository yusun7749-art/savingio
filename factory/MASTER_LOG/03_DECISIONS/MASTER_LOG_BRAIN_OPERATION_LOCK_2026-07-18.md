# MASTER LOG BRAIN OPERATION LOCK

## 2026-07-18 최우선 운영 규칙

## MASTER LOG = 리나의 뇌(State)

MASTER LOG는 단순 기록장이 아니다.
MASTER LOG는 단순 인수인계 문서가 아니다.

MASTER LOG는 Savingio 프로젝트의 현재 상태(State)를 복원하는 리나의 뇌다.

새 채팅, 새 세션, 새 작업 환경에서도 MASTER LOG를 기준으로 직전 상태를 복원해야 한다.

---

## 답변 완료 = MASTER LOG 갱신 완료

하나의 작업 단위는 다음 순서로 완료한다.

1. 실제 작업 수행
2. 실제 검증
3. 사용자에게 결과 보고
4. MASTER LOG 기록
5. 다음 작업 위치 갱신

MASTER LOG 기록이 빠지면 작업 완료로 판단하지 않는다.

---

## 실시간 기록 원칙

프로젝트 종료 후 한 번에 작성하지 않는다.

하루 종료 후 작성하지 않는다.

기억나는 내용을 모아서 작성하지 않는다.

사용자의 작업 명령:

- 진행
- 다음
- 시작
- 0
- 이어서 진행

등 하나의 작업 단위가 끝날 때마다 즉시 기록한다.

---

## 기록 항목

모든 작업 기록은 시간순으로 남긴다.

필수 항목:

- 시간
- 사용자 지시
- 실제 수행 내용
- 수정 파일
- 성공 결과
- 실패 결과
- 실패 원인
- 결정 사항
- 다음 작업

---

## 실행 우선 원칙

실제 확인 전 추측으로 판단하지 않는다.

금지:

- GitHub 연결이 안 된다고 단정
- 쓰기 권한이 없다고 단정
- 저장할 수 없다고 단정
- 도구 사용 불가라고 단정

반드시:

확인 → 실행 → 결과 → 보고

순서를 따른다.

---

## 새 채팅 복구 규칙

'마스터로그 확인하고 와'는 다음 절차를 의미한다.

1. MASTER_LOG_CURRENT 확인
2. MASTER LOG 전체 확인
3. 01_SUCCESS~06_QA 확인
4. 최신 main 상태 확인
5. 차이 검증
6. 마지막 기록 다음 작업부터 시작

사용자가 과거 내용을 다시 설명하게 만들지 않는다.
