## 최우선 결정

Savingio 기존 글 교체 작업은 **글쓰기 헌법을 최우선**으로 적용한다.

글을 새로 작성하거나 품질을 보강할 때 기존 페이지의 틀을 임의로 재설계하지 않는다. 사용자가 승인한 공통 템플릿 안에서 **본문 콘텐츠만 교체**한다.

## 절대 변경 금지

- 기존 3분할 페이지 구조
- 왼쪽 Site Explorer·Brain Navigation·검색 트리
- 중앙 본문 폭과 배치
- 오른쪽 고정 템플릿의 구성과 정보량
- 기존 공통 CSS와 공통 클래스 계보
- Header와 Footer
- 카테고리 탐색 구조
- 사용자가 승인한 카드 배열과 공통 UI

새 글을 작성한다는 이유로 인라인 CSS를 새로 만들거나 별도 레이아웃을 삽입하지 않는다.

## 글 교체 허용 범위

- H1과 Lead
- 작성·검수 정보
- 5초 결론
- 30초 질문
- 목차
- 문제 해결형 본문
- 비교표와 확인표
- 실행 체크리스트
- FAQ와 FAQ 구조화 데이터
- 공식 근거
- 관련 글 링크
- SEO title·description·canonical·Article schema

단, 위 콘텐츠도 기존 템플릿의 위치와 클래스 안에서만 교체한다.

## 고정 콘텐츠 DNA

Breadcrumb
→ H1
→ Lead
→ 작성·검수
→ 5초 결론
→ 30초 질문
→ 목차
→ 본문
→ 표
→ 체크리스트
→ FAQ
→ 관련글
→ Footer

## 본문 썸네일 LOCK

본문 상단 `<figure class="thumb">`는 삭제 상태를 유지한다.

목적:

- Above The Fold 확보
- 첫 광고 위치 확보
- 페이지 시작 속도 개선
- 불필요한 본문 밀림 방지

## 작업 단위

- 1턴에 글 1개만 수정한다.
- 한 글을 완료하고 실제 운영 화면을 검수하기 전 다음 글로 이동하지 않는다.
- URL은 유지한다.
- 기존 짧은 원고는 부분 덧씌우기가 아니라 문제 해결형 본문으로 전면 재작성할 수 있다.
- 다만 틀·좌측·우측·공통 CSS는 건드리지 않는다.

## 재발 방지 QA

수정 전:

1. 대상 글의 기존 HTML을 전체 확인한다.
2. 동일 템플릿을 사용하는 승인 완료 글을 확인한다.
3. 좌측·중앙·우측·Header·Footer 클래스와 콘텐츠를 구분한다.
4. 실제 교체 범위를 본문 콘텐츠로 제한한다.

수정 후:

1. 기존 좌측 탐색 구조 보존 확인
2. 기존 우측 템플릿 구성과 정보량 보존 확인
3. 새 인라인 페이지 레이아웃 CSS 삽입 여부 검사
4. 콘텐츠 DNA 순서 확인
5. H1 1개 확인
6. 본문 상단 figure 썸네일 없음 확인
7. 내부 링크와 카테고리 링크 확인
8. 운영 화면에서 3분할 구조와 모바일 화면 확인

위 항목 중 하나라도 실패하면 PASS를 출력하지 않는다.
