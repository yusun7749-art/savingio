# MASTER VISION — LINA OS 글로벌 풀 자동화 사업 방향

최종 갱신: 2026-07-24 KST

---

## 0. 이 문서가 필요한 이유

이 문서는 단순한 아이디어 메모가 아니다.

향후 새로운 대화, 새로운 AI, 새로운 작업자, 새로운 개발 단계에서 프로젝트가 다른 방향으로 새지 않도록 하기 위한 최상위 방향 문서다.

짧게 “자동화한다”, “글로벌로 간다”, “콘텐츠를 만든다”라고만 기록하면 다음과 같은 오해가 생길 수 있다.

- 단순 글 자동 작성 도구로 축소될 수 있다.
- 유튜브 업로드 도구 하나로 오해할 수 있다.
- 번역 기능만 추가하면 글로벌 시스템이라고 판단할 수 있다.
- Savingio 자체를 최종 목적이라고 착각할 수 있다.
- 사용자가 계속 API, Git, 서버, 인증, 배포 과정을 직접 처리하는 개발자형 구조로 돌아갈 수 있다.
- 콘텐츠 생산은 자동화했지만 상품 연결, 수익 추적, 성과 분석, 다음 전략까지 이어지지 않을 수 있다.

따라서 이 문서는 프로젝트의 출발 배경, 사용자의 문제, 제품 철학, 국내 테스트의 의미, 글로벌 확장 구조, 제휴 수익 모델, 사용자 화면, 관리자 화면, 자동화 범위, 안전장치, 금지 방향, 단계별 개발 원칙을 상세히 고정한다.

## 1. 사용자가 해결하려는 실제 문제

사용자는 개발자가 아니다.

사용자가 원하는 것은 다음과 같은 반복 설명과 수동 조작이 아니다.

- “이 메뉴로 들어가세요.”
- “이 버튼을 누르세요.”
- “API 키를 발급하세요.”
- “OAuth를 연결하세요.”
- “Git에 커밋하고 푸시하세요.”
- “Cloudflare 설정을 확인하세요.”
- “Python을 설치하세요.”
- “환경변수 파일을 수정하세요.”
- “영상 파일을 만든 뒤 다시 업로드하세요.”

이런 안내가 시작되는 순간 사용자는 머리가 아프고, 대부분의 일반 사용자도 같은 지점에서 자동화 사업을 포기한다.

즉, 사람들이 AI 부업과 자동화 사업을 포기하는 핵심 이유는 AI가 부족해서가 아니라, 결국 중간 과정에서 사용자가 개발자·편집자·업로더·운영자가 되어야 하기 때문이다.

리나 OS는 이 문제를 제거해야 한다.

## 2. 프로젝트의 최종 정의

리나 OS는 단순한 AI 도구가 아니다.

리나 OS는 컴퓨터를 잘 모르는 일반인도 AI 기반 콘텐츠 회사를 운영할 수 있도록 만드는 통합 운영체제다.

최종 목표는 사용자가 복잡한 기술을 배우는 것이 아니라, 다음과 같이 단순하게 사용하는 것이다.

1. 리나 실행
2. 운영할 브랜드 또는 채널 선택
3. “자동 운영 시작” 클릭
4. 결과 확인
5. 필요할 때만 승인

나머지 과정은 시스템이 처리한다.

리나 OS의 핵심 정의:

> 주제 발굴부터 콘텐츠 제작, 다국어 변환, 영상·이미지 제작, 플랫폼 배포, 상품 연결, 수익 추적, 성과 분석, 다음 콘텐츠 개선까지 하나의 흐름으로 자동 수행하는 AI 운영체제

## 3. 제품 철학

### 3.1 개발자를 위한 프로그램이 아니다

사용자가 Git, API, OAuth, Python, Docker, 서버, 데이터베이스, FFmpeg, 환경변수, 웹훅 구조를 이해해야 한다면 제품 설계가 실패한 것이다.

기술 요소는 내부에 존재할 수 있지만 사용자에게 노출되면 안 된다.

### 3.2 부모님도 설명 없이 사용할 수 있어야 한다

제품의 가장 중요한 성공 기준은 다음 질문이다.

> 우리 부모님이 별도의 설명서를 읽지 않고도 사용할 수 있는가?

답이 아니면 다시 설계한다.

### 3.3 버튼은 최소, 자동화는 최대

일반 사용자 화면은 단순해야 한다.

- 연결 상태
- 오늘 생성 예정 콘텐츠 수
- 오늘 배포 예정 플랫폼
- 자동 운영 시작
- 오류 또는 승인 요청

이 정도만 보여야 한다.

### 3.4 사용자는 제작자가 아니라 운영 승인자다

사용자가 매번 글을 쓰고, 이미지를 고르고, 영상을 편집하고, 제목을 붙이고, 링크를 복사하고, 업로드하는 구조는 목표가 아니다.

사용자는 사업 방향을 정하고, 위험하거나 중요한 항목만 승인하는 역할을 맡는다.

### 3.5 설명서 없이 사용할 수 있어야 한다

“어떻게 해야 하지?”라는 생각이 들지 않도록 화면과 흐름을 설계한다.

## 4. 왜 풀 자동화가 필수인가

이 사업은 부분 자동화로는 성립하기 어렵다.

글만 자동으로 작성되고 나머지 작업을 사람이 처리하면 새로운 노동이 된다.

영상만 자동 생성되고 업로드, 제목, 설명, 링크, 분석을 사람이 맡아도 부업이 아니라 반복 업무가 된다.

따라서 다음 흐름이 처음부터 끝까지 연결되어야 한다.

1. 트렌드 및 검색 수요 분석
2. 오늘 제작할 주제 선정
3. 사실 확인 및 출처 검토
4. 콘텐츠 기획
5. 글 작성
6. 영상 대본 작성
7. 이미지 생성
8. 영상 생성 및 편집
9. 음성 생성
10. 자막 생성
11. 배경음악 및 음량 검수
12. 썸네일 생성
13. 제목·설명·태그 작성
14. SEO 구조 작성
15. 국가별 언어 변환
16. 국가별 판매 가능 상품 탐색
17. 국가별 제휴 링크 연결
18. 사이트 게시
19. 유튜브·쇼츠·SNS 예약 업로드
20. 게시 후 링크 검증
21. 조회수·클릭·전환·수익 수집
22. 성과 분석
23. 다음 콘텐츠 전략 자동 수정

이 흐름 중 사람이 반복적으로 개입해야 하는 단계가 많으면 풀 자동화가 아니다.

## 5. Savingio의 정확한 역할

Savingio는 최종 목적이 아니다.

Savingio는 리나 OS의 첫 번째 실제 운영 브랜드이자 국내 테스트베드다.

Savingio에서 먼저 검증할 항목:

- 콘텐츠 자동 생성 품질
- SEO 구조
- 검색 유입
- 애드센스 승인 및 광고 수익
- 내부 링크 구조
- 계산기 및 도구 연결
- 제휴 상품 연결
- 사이트 게시 자동화
- 관리자 승인 흐름
- 오류 감지 및 복구
- 수익 데이터 수집
- 콘텐츠 성과 분석

Savingio에서 성공적으로 검증된 기능은 리나 OS의 공통 DNA로 편입한다.

즉, Savingio는 하나의 사이트이면서 동시에 다음 브랜드를 만들기 위한 실험실이다.

향후 K-Beauty, K-Food, Travel, Parenting, Living, Education 등의 브랜드를 추가해도 같은 엔진을 사용할 수 있어야 한다.

## 6. 국내 시장의 의미

국내 시장은 최종 목표 시장이 아니다.

국내는 시스템을 검증하기 가장 좋은 첫 번째 시장이다.

국내 테스트의 목적:

- 언어와 문화 이해가 쉬운 환경에서 품질 검증
- 사용자 반응 확인
- SEO 및 색인 확인
- 콘텐츠 자동 생성 오류 수정
- 애드센스와 제휴 수익 흐름 검증
- 관리자 화면과 승인 흐름 검증
- 배포 및 복구 시스템 검증
- 실제 수익 발생 여부 검증

국내에서 검증해야 할 질문:

- 자동 생성 결과가 사람이 읽을 수 있는 수준인가?
- 글과 영상의 품질이 일정한가?
- 중복·오류·저작권 문제가 자동으로 걸러지는가?
- 검색 유입이 발생하는가?
- 제휴 링크가 실제로 작동하는가?
- 클릭과 구매가 추적되는가?
- 수익이 실제로 발생하는가?
- 장애가 나면 시스템이 복구하거나 알림을 보내는가?

이 검증을 통과한 뒤 글로벌 확장으로 넘어간다.

---

## 7. 글로벌 확장의 이유

국내 시장은 규모가 제한적이다.

특히 K-Beauty, K-Food, K-Travel, K-Lifestyle 등은 국내보다 해외에서 더 큰 수요와 관심을 받을 수 있다.

사용자의 핵심 철학:

> 웅덩이에서 노느니 풍천에서 논다.

이 말은 단순히 해외에 진출한다는 뜻이 아니다.

다음 원칙을 의미한다.

- 처음부터 작은 국내 시장만을 기준으로 시스템을 설계하지 않는다.
- 국내는 검증 장소이고, 최종 구조는 세계 시장을 향한다.
- 특정 국가나 특정 플랫폼에 종속되지 않는 엔진을 만든다.
- 하나의 콘텐츠가 언어와 플랫폼을 바꾸어 여러 국가에서 수익을 만들 수 있어야 한다.
- 동일 상품이라도 국가별 구매 가능 판매처와 제휴 링크를 다르게 연결한다.
- 글로벌 확장은 번역만 하는 것이 아니라 국가별 시장·플랫폼·구매 구조를 반영해야 한다.

## 8. 글로벌 수익 모델

### 8.1 직접 판매자가 되지 않는다

사용자는 쇼핑몰 운영자가 되는 것을 목표로 하지 않는다.

원칙적으로 다음을 직접 담당하지 않는다.

- 상품 제조
- 재고 보관
- 포장
- 해외 배송
- 반품
- 환불
- 고객 상담
- 판매 후 CS

이 업무는 해당 브랜드, 판매처, 제휴 플랫폼이 담당한다.

### 8.2 우리의 역할

우리의 역할은 콘텐츠와 구매 연결이다.

1. 소비자가 궁금해할 콘텐츠 제작
2. 신뢰 가능한 정보 제공
3. 관련 상품 소개
4. 소비자의 국가에서 실제 구매 가능한 판매처 연결
5. 제휴 링크를 통한 구매 전환
6. 발생한 제휴 수익 수취

### 8.3 가능한 수익원

- 애드센스 및 디스플레이 광고
- 국내 제휴 마케팅
- 해외 Affiliate Program
- 영상 플랫폼 광고 수익
- 브랜드 협찬
- 스폰서 콘텐츠
- 리드 생성 수익
- 디지털 콘텐츠 및 도구 수익
- 향후 리나 OS 사용료 또는 SaaS 구독료

### 8.4 국가별 상품 연결

예: 한국 선크림 콘텐츠

- 한국 사용자: 국내 판매처 또는 국내 제휴 링크
- 미국 사용자: 미국에서 배송 가능한 판매처 및 미국 제휴 링크
- 일본 사용자: 일본 판매처 및 일본 제휴 링크
- 유럽 사용자: 유럽 배송 지원 판매처 및 해당 지역 제휴 링크

따라서 하나의 고정 링크를 전 세계에 사용하는 방식이 아니라, 사용자 국가와 판매 가능 여부를 고려하는 수익 엔진이 필요하다.

## 10. 사용자 화면 원칙

- OAuth Token
- Client Secret
- JSON
- Webhook
- Cron
- Environment Variable
- API Endpoint
- Git Commit

## 12. 승인센터의 역할

초기에는 무조건 전면 무인 발행을 하지 않는다.

잘못된 정보, 중복 콘텐츠, 저작권 문제, 상품 링크 오류가 그대로 발행되는 위험을 막아야 한다.

### 1단계: 원클릭 승인형 자동화

시스템이 모든 결과물을 만들고, 사용자는 한 번의 승인으로 배포한다.

### 2단계: 조건부 자동 발행

QA 기준을 통과한 콘텐츠는 자동 발행하고, 위험 항목만 사용자에게 승인 요청한다.

### 3단계: 무인 운영

주제 선정부터 제작, 배포, 분석까지 자동 수행하고 문제가 발생했을 때만 사용자에게 알린다.

승인센터는 자동화를 방해하는 장치가 아니라, 풀 자동화로 안전하게 넘어가기 위한 핵심 관문이다.

## 14. 절대 금지 방향

다음 방향으로 돌아가지 않는다.

### 14.1 사용자가 개발 작업을 반복하는 구조

사용자에게 매번 파일 수정, Git 명령, API 설정, 서버 접속을 요구하지 않는다.

### 14.2 자동화처럼 보이지만 사람이 마무리하는 구조

글만 생성하고 업로드는 사람이 하는 방식, 영상만 생성하고 제목과 링크는 사람이 붙이는 방식은 최종 목표가 아니다.

### 14.3 Savingio만을 위한 일회성 프로그램

모든 기능은 향후 다른 브랜드와 국가에 재사용할 수 있도록 공통 엔진으로 설계한다.

### 14.4 국내 시장에만 맞춘 구조

한국어, 국내 링크, 국내 플랫폼만 하드코딩하지 않는다.

### 14.5 번역만으로 글로벌이라고 판단하는 것

글로벌화는 언어, 문화, 플랫폼, 판매처, 배송 가능 국가, 제휴 조건, 법률 표시까지 포함한다.

### 14.6 판매·재고·CS 사업으로 방향 전환

사용자가 직접 상품을 사입하고 배송하며 CS를 처리하는 쇼핑몰 모델을 기본 방향으로 삼지 않는다.

### 14.7 무검수 자동 대량 발행

QA와 안전장치 없이 콘텐츠를 무제한 자동 발행하지 않는다.

### 14.8 진행 상황을 코드로만 보여주는 구조

사용자가 눈으로 제목, 영상, 상품, 승인 상태, 배포 상태, 수익 상태를 확인할 수 있어야 한다.

## 18. 최종 목표 문장

> 컴퓨터를 잘 모르는 사람도 버튼 한 번으로 AI 회사를 운영할 수 있게 한다.

> 하나의 콘텐츠가 여러 언어와 플랫폼으로 확장되고, 각 국가에서 실제 구매 가능한 상품과 자동 연결되어 수익을 만들게 한다.

> 판매, 재고, 배송, 반품, CS는 판매업체가 담당하고, 우리는 콘텐츠와 연결을 통해 광고 및 제휴 수익을 얻는다.

> Savingio는 이 거대한 시스템의 첫 번째 국내 테스트베드이며, 국내 검증 후 세계 시장으로 확장한다.

> 웅덩이에서 노느니 풍천에서 논다.

## 19. 새 대화·새 AI 인수인계 지시

새로운 AI 또는 작업자가 이 문서를 읽었을 때 다음 내용을 절대 초기화하지 않는다.

- 사용자는 개발자가 아니다.
- 사용자가 반복적으로 기술 설정을 수행하게 해서는 안 된다.
- 최종 목표는 부분 자동화가 아니라 풀 자동화다.
- Savingio는 최종 목적이 아니라 첫 번째 검증 브랜드다.
- 국내는 최종 시장이 아니라 테스트 시장이다.
- 글로벌 확장은 번역 기능만 의미하지 않는다.
- 사용자는 직접 판매·재고·배송·CS를 담당하지 않는다.
- 수익은 콘텐츠, 광고, 제휴 연결을 중심으로 설계한다.
- 관리자 페이지는 사용자가 전체 진행 상황을 일반 업무 언어로 확인하고 승인하기 위한 운영본부다.
- 시스템은 향후 여러 브랜드, 국가, 플랫폼에 확장 가능해야 한다.

새 기능을 제안하거나 개발할 때 반드시 다음 질문을 먼저 한다.

1. 이 기능이 사용자의 반복 작업을 줄이는가?
2. 이 기능이 풀 자동화에 가까워지게 하는가?
3. 일반인이 설명 없이 사용할 수 있는가?
4. Savingio 외 다른 브랜드에도 재사용할 수 있는가?
5. 국내 검증 후 글로벌 확장이 가능한 구조인가?
6. 사용자가 판매·배송·CS 업무를 떠안게 만들지 않는가?
7. 사용자에게 기술 용어가 아니라 실제 업무 상태로 보이는가?

하나라도 충족하지 못하면 방향을 재검토한다.
