# Savingio Glass Explorer V1

## 목적
왼쪽 영역을 단순 카테고리 메뉴가 아니라 현재 위치와 다음 탐색 방향을 알려주는 생활 문제 탐색기로 만든다.

## 핵심 원칙
1. 분류명은 실제 포함 글과 일치할 때만 노출한다.
2. 사용자의 상황 문장은 검색과 추천 문구에 사용하고, 분류명 자체는 객관적인 주제명으로 유지한다.
3. 현재 글이 포함된 대분류·중분류·소분류만 자동으로 펼친다.
4. 나머지 분류는 한 단계만 보이고 불필요한 글 목록은 접힌 상태를 유지한다.
5. 현재 글은 색상만이 아니라 왼쪽 강조선과 `현재 글` 표시로 구분한다.
6. 글 개수 숫자는 탐색에 필요하지 않으면 노출하지 않는다.
7. Explorer는 본문보다 시각적으로 강하지 않아야 한다.
8. 모바일에서는 전체 화면을 가리는 메뉴가 아니라 닫기 쉬운 슬라이드 패널로 동작한다.

## 화면 구조
- 브랜드 라벨: SAVINGIO EXPLORER
- 제목: 생활 문제 찾기
- 안내: 지금 겪는 일을 검색하거나 주제를 선택하세요.
- 검색창
- 대분류 카드
- 중분류 목록
- 소분류 목록
- 현재 글 강조

## 시각 규칙
- 따뜻한 크림색 반투명 배경
- 20px 이상 둥근 모서리
- 얇은 흰색 테두리
- 약한 골드 포인트
- 과도한 아이콘과 장식 금지
- 긴 제목은 2줄까지만 표시
- 글자 크기는 최소 12px 이상

## 동작 규칙
- 현재 경로가 있는 대분류는 자동 펼침
- 다른 대분류를 열면 현재 경로가 아닌 열린 대분류는 자동 닫힘
- 검색 중에는 일치하는 경로만 노출
- 검색 결과가 없으면 입력 예시 제공
- 깨진 URL, `tokens truncated`, 비정상적으로 긴 href는 렌더링하지 않음

## 단계별 적용
### V1
- Glass UI 정리
- 현재 위치 강조
- 가독성·여백·클릭 영역 개선
- 글 개수 표시 제거

### V2
- Content Taxonomy 감사 결과와 연결
- 분류 불일치 글 자동 제외
- 현재 글 주변의 다음 읽을 글 3개만 표시

### V3
- 사용자 상황 검색어와 객관적 분류를 연결하는 별도 매핑 레이어
- 계산기와 관련 제도 연결
- 배포 전 Explorer QA 자동화
