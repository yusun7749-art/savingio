# Savingio Article DNA V2.3 — 성형틀

## 현재 단계
기존 글 전체를 수정하지 않는다. 먼저 이 DNA와 자동 QA를 완성하고, 서로 다른 글 유형의 비공개 테스트를 통과한 뒤 대표 승인 후 기존 글 재정비를 시작한다.

## 고정 독자 행동 흐름
1. Hero — 사용자가 이 글을 찾은 실제 이유와 현재 상태를 3초 안에 보여준다.
2. Quick Answer — 가장 궁금한 답과 핵심 요약을 먼저 준다.
3. Calculator / Check Tool — 사용자가 아는 사실만 입력하거나 바로 확인한다.
4. Result — 결과가 왜 그렇게 나왔는지 해석한다.
5. Deep Information — 사례·비교·표·주의사항·공식기준으로 궁금증을 깊게 해결한다.
6. Situation — 직장인·프리랜서·계약직·아르바이트 등 상황별 차이를 분기한다.
7. Action — 지금부터 실제로 할 일을 순서대로 제시한다.
8. Official — 공식기관과 원문 기준을 확인한다.
9. FAQ — 읽은 뒤에도 남을 질문을 해결한다.
10. Related — 관련 계산기와 관련 글을 연결한다.
11. Next — 다음 궁금증과 다음 행동으로 자연스럽게 이동한다.

## 글 유형 6종
- 계산·결과형
- 자격·신청형
- 절약·실천형
- 비교·선택형
- 문제 해결형
- 건강·안전 안내형

각 유형은 `data/article-type-rules.json`의 흐름·필수 증거·최소 QA 기준을 따른다. 공통 레이아웃은 유지하되, 내용 순서와 판단 포인트는 유형별로 달라진다.

## Navigation DNA
- 본문 목차는 한 줄 가로 스크롤형으로 고정한다.
- 현재 읽는 섹션은 자동 강조한다.
- 데스크톱 왼쪽에는 대분류 → 중분류 → 소분류 탐색기를 둔다.
- 모바일에서는 `주제 탐색` 버튼으로 같은 탐색기를 연다.
- 현재 페이지가 탐색기에 있으면 해당 대분류를 자동으로 펼치고 현재 항목을 표시한다.
- 오른쪽에는 현재 글과 직접 연결되는 계산기와 다음 글만 배치한다.

## Curiosity DNA
각 핵심 단락은 결론으로 닫지 않고 현재 답에서 자연스럽게 생기는 다음 질문으로 연결한다. 반복 문구, 빈 질문, 억지 연결은 QA 실패로 처리한다.

## Content Density 기준
- 목표 분량: 공백 제외 3,000~5,000자
- 요약 카드 4개
- 핵심 정보 섹션 6개
- 상황별 사례: 유형 기준 이상
- 비교표 1개 이상
- 체크리스트 7개 이상
- FAQ 5개 이상
- 공식기관 1개 이상
- 관련 계산기 2개 이상
- 관련 정보글 2개 이상
- 결정 기준: 유형 기준 이상
- 검색 의도와 독자 상태가 구체적으로 분리되어야 함

## 자동 QA
- 필수 필드와 글 유형 검사
- 분량·사례·행동·결정 기준·표·체크리스트·FAQ 검사
- 검색 의도·독자 상태의 얕은 문장 검사
- FAQ·다음 질문·도입부 중복 검사
- 고정형 궁금증 문구 반복 검사
- 내부 링크와 탐색기 링크 존재 검사
- 템플릿 토큰 잔존 검사
- QA 실패 시 기존 HTML을 덮어쓰지 않고 종료 코드 1 반환
- QA 성공 시에만 임시 빌드 결과를 실제 articles 폴더로 반영

## 운영 파일
- 콘텐츠: `data/article-configs-v2.json`
- 글 유형: `data/article-type-rules.json`
- 사이트 탐색: `data/site-navigation.json`
- 공통 HTML: `templates/article-v2.html`
- 빌더: `tools/build-articles-v2.py`
- 비공개 유형 테스트: `tests/article-type-fixtures.json`
- 검증: `tools/verify-article-dna-v2.py`

## 승인 순서
성형 → 비공개 유형 테스트 → 샘플 화면 검수 → 대표 승인 → LOCK → 기존 글 일괄 재정비
